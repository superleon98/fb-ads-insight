# FB 广告洞察系统 - 部署指南

## 一、你需要准备什么

| 项目 | 用途 | 费用 |
|------|------|------|
| 一台云服务器（推荐） | 运行系统 | 阿里云/腾讯云 ~99元/年 |
| AdLibrary.com API Key | 抓取广告数据 | ~€299/月（Business 计划） |

> 如果没有服务器，也可以在自己电脑上先用 Docker 跑起来，团队内网访问。

## 二、API Key 申请步骤

### 2.1 AdLibrary.com API Key（必须）

这是抓取 Meta/Google/TikTok 广告数据的核心。

1. 访问 https://adlibrary.com
2. 点击右上角 **Sign Up** 注册账号
3. 升级到 **Business** 计划（才有 API 权限）
4. 进入 **Developer Dashboard**
5. 点击 **Create API Key**
6. 复制生成的 Key（格式：`adl_xxxxxxxx`）

把这个 Key 填到 `.env` 文件的这一行：
```
ADLIBRARY_API_KEY=adl_你的真实_key
```

### 2.2 Kimi API Key（必须）

这是 AI 分析广告的功能。

1. 访问 https://platform.moonshot.cn
2. 用手机号注册账号
3. 进入 **API Key 管理** → **新建 API Key**
4. 复制 Key（格式：`sk-xxxxxxxx`）

填到 `.env`：
```
KIMI_API_KEY=sk-你的真实_key
```

### 2.3 飞书 SSO（可选）

如果不需要团队登录功能，可以跳过。不配置飞书时，AI 分析功能直接可用。

1. 访问 https://open.feishu.cn
2. 点击 **创建企业自建应用**
3. 记录 **App ID** 和 **App Secret**
4. 左侧菜单 **安全设置** → 添加重定向 URL：
   ```
   http://你的服务器IP/login/callback
   ```
5. 左侧菜单 **权限管理** → 添加权限：
   - `authen:basic`（获取用户基本信息）
6. **版本管理与发布** → 创建版本 → 申请发布

填到 `.env`：
```
FEISHU_APP_ID=cli_你的_app_id
FEISHU_APP_SECRET=你的_app_secret
FEISHU_REDIRECT_URI=http://你的服务器IP/login/callback
```

## 三、部署方式（二选一）

### 方式 A：Docker 一键部署（推荐，最简单）

**前提**：服务器已安装 Docker 和 Docker Compose

```bash
# 1. 把代码上传到服务器（假设上传到 /opt/fb-ads 目录）
cd /opt/fb-ads

# 2. 编辑环境变量，填入你的 API Key
vi .env
# 按 i 进入编辑模式，修改 ADLIBRARY_API_KEY 和 KIMI_API_KEY
# 按 Esc，输入 :wq 保存

# 3. 一键启动
sudo docker-compose up -d

# 4. 查看状态
sudo docker-compose ps

# 5. 查看日志
sudo docker-compose logs -f backend
```

启动后访问：`http://你的服务器IP`

**更新代码时**：
```bash
cd /opt/fb-ads
sudo docker-compose down
sudo docker-compose up -d --build
```

### 方式 B：手动部署（适合熟悉 Node.js 的环境）

**3.1 部署后端**

```bash
cd server
npm install
# 复制 .env 文件到 server 目录
cp ../.env .
npm run build
npm start
```

**3.2 部署前端**

前端是纯静态文件，可以用 Nginx、Apache 或任何静态托管服务。

```bash
# 把 app/dist 文件夹放到 Nginx 的 web 目录
cp -r app/dist /var/www/fb-ads
```

Nginx 配置参考 `nginx.conf` 文件。

## 五、常见问题

### Q1：API Key 填到哪里？

**只需要修改一个文件：`.env`**

找到这两行，替换掉示例值：
```env
ADLIBRARY_API_KEY=adl_你的真实_key
KIMI_API_KEY=sk-你的真实_key
```

### Q2：如何确认后端正常启动？

```bash
curl http://localhost:3001/api/ads/health
```

正常应该返回：
```json
{"status":"ok","service":"fb-ads-insight-server","version":"2.0.0"}
```

### Q3：前端访问不了后端？

1. 检查防火墙是否放行了 3001 端口
2. 确认 `.env` 里的 `CORS_ORIGIN` 包含你的前端域名
3. 如果用 docker-compose，Nginx 会自动代理 API 请求，无需额外配置

### Q4：不想用飞书登录怎么办？

不配置 `FEISHU_APP_ID` 即可。系统会自动跳过登录验证，AI 分析功能直接可用。

### Q5：如何更新代码？

```bash
cd /opt/fb-ads
git pull  # 或者用 FTP/SCP 上传新代码
sudo docker-compose down
sudo docker-compose up -d --build
```
